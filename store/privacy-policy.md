# Privacy Policy — stikfix

**Effective date:** 2026-06-17
**Contact:** omernesher@gmail.com

---

## Summary

stikfix collects no personal data, sends no data to any remote server, and performs no analytics or telemetry. All data generated by the extension (note text, screenshots, page metadata) is written exclusively to the user's own disk via a local authenticated connection. Nothing leaves the user's machine.

---

## What stikfix does with your data

### Data generated

When you drop a sticky note, stikfix captures:
- The text of your note
- The URL and title of the current page
- A timestamp
- Viewport dimensions
- A screenshot of the visible tab (or a cropped region you select)
- If you anchor to an element: the element's CSS selector, a subset of its computed styles, a portion of its outerHTML, its bounding box, data-* attributes, ARIA role and label, and a best-effort React component name

### Where data goes

All captured data is sent only to **your own localhost host** (the `stikfix` npm package running on your machine, bound to `127.0.0.1`). The connection is authenticated with a token that is exchanged through the operating system's Native Messaging API — the token is never transmitted over a network. The host writes note content as `.md` files and screenshots as `.png` files into the project folder you specified at setup (`npx stikfix init --root <folder>`).

**No data is sent to any remote server, cloud service, or third party at any time.**

### Data not collected

- No personal information (name, email, account credentials)
- No browsing history
- No analytics events or usage telemetry
- No crash reports or error logging to any external service
- No advertising identifiers

## Permissions — plain-language explanation

The extension requests the following browser permissions:

| Permission | Why it is needed |
|---|---|
| `activeTab` | Captures a screenshot of the current tab when you drop a note. Access is limited to the tab you are actively using and only while Review Mode is active. |
| `scripting` | Injects the Review Mode UI (toolbar, pins, notes panel) into the current page on demand. The UI is injected only when you click "Enter Review Mode" — never automatically on page load. |
| `storage` | Stores your per-origin folder preferences (which project folder maps to which website origin) and your pairing state locally in the browser. No data is synced to any cloud. |
| `tabs` | Detects URL changes in single-page apps so that pins update when you navigate within the same tab. |
| `nativeMessaging` | Communicates with the stikfix companion host running on your machine via the OS Native Messaging API. This is the only channel through which notes reach disk. |
| `http://127.0.0.1/*` and `http://localhost/*` | Allows the extension to make HTTP requests to the companion host running on your machine. Required for note writes, reads, and status updates. |
| `<all_urls>` (optional) | Allows the Review Mode content script to be injected into any page. This permission is **not** requested at install time — it is requested on demand the first time you click "Enter Review Mode" on a page. You can decline it; if declined, Review Mode will not function on that page. |

## Data retention and deletion

Notes and screenshots are stored as plain files on your disk in the folder you chose. You can view, edit, or delete them at any time using any file manager or text editor. Uninstalling the extension or the host does not delete any notes you have previously saved — they remain on disk until you delete them manually. Browser storage (folder preferences, pairing state) is cleared when you uninstall the extension.

---

## Third parties

stikfix shares no data with any third party. The extension makes no network requests to any server other than `127.0.0.1` and `localhost`. The companion host makes no outbound network requests of any kind.
