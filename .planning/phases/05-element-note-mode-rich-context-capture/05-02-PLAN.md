<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page → content script | mousemove/click `e.target` is a page-controlled element; its tag/id/class strings are untrusted |
| content script ↔ shadow-root UI | overlay/label live in the shadow root; must never be the picker target |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering (XSS) | hover label text | mitigate | Label prefix (tag/id/class) set via `textContent` only — never innerHTML. Enforced by grep gate in Task 1 acceptance. |
| T-05-06 | Tampering | finder called on shadow element | mitigate | mousemove guards `container.contains(target)` so sfx-internal elements are never picked (Pitfall 5); finder itself runs in Plan 03 on page elements only. |
| T-05-07 | Spoofing | page forges picker click | accept | `e.target` is set by the browser's event dispatch; a page script cannot forge an arbitrary target to a real document click (RESEARCH security domain). Low risk, developer-only tool. |
| T-05-08 | — | pick mode click also hits page handler | accept | UI-SPEC: not calling stopPropagation is intentional for v1; page may handle its own click. Documented; aggressive-capture pages are a Phase 8 edge case (RESEARCH Open Q3). |
</threat_model>
