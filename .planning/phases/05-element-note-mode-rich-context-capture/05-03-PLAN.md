| Boundary | Description |
|----------|-------------|
| picked page element → ElementContext → card DOM | selector/text/outerHTML/attrs/component name are untrusted page strings entering the shadow UI and the payload |
| content script → SW (SFX_CAPTURE_TAB / SFX_SEND_ANNOTATION) | privileged capture + host POST cross here; SW is sole privileged caller |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering (XSS) | context header (.sfx-ctx-header-text) | mitigate | `buildContextSummary` output set via `textContent` ONLY (the prime page-derived risk surface, RESEARCH security domain). No innerHTML in card.ts — grep gate in Task 1 acceptance. |
| T-05-10 | IDOR / origin-trust | captureTab / SFX_SEND_ANNOTATION relay | mitigate | Reuse the existing SW sender-binding IDOR guard (handleCaptureTab checks sender.tab.id === reqTabId, commit 6736413) and the existing SEND_ANNOTATION relay — content script supplies payload only; SW resolves tab/window/host itself. No new privileged message added. |
| T-05-11 | Information disclosure | outerHTML may contain sensitive page data | accept | Developer-only local tool; outerHTML truncated to 2000 chars, written as inert text data by host (not executed). Exhaustive redaction is out of scope (RESEARCH Pitfall 6 / Phase 8). |

| T-05-13 | Information disclosure | own UI leaks into +1.png | mitigate | Strict hide-all → waitTwoRafs → captureTab order (Pitfall 3); box drawn onto the canvas AFTER capture (D-02a), never the live overlay. Verified in the checkpoint. |
