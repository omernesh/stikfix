<objective>
Wire the four Plan 01 modules into a real `node:http` server (`server.ts`) and a CLI boot
sequence (`index.ts`) that binds `127.0.0.1` on a free port in 39240-39260, serves `GET /status`,
handles `OPTIONS` preflight, and processes token-gated `POST /annotation` end-to-end —
plus an integration test that boots the server and exercises every route.

Purpose: This is the vertical slice that makes the host real — after this plan a developer can
run `npm run host -- --root <dir>`, see the startup line, GET /status, and POST an annotation
that lands as a .md file on disk. It connects the tested-in-isolation modules into a running server.
Output: `host/src/server.ts`, the rewritten `host/src/index.ts`, and `host/test/server.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN -> listening socket | Binding a non-loopback interface would expose the file-writing endpoint to other hosts |
| browser page -> POST /annotation | Any page guessing the port could write files without the token gate |
| oversized body -> server memory | Base64 screenshot bodies could exhaust memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-bind | Elevation of Privilege | index.ts listen | mitigate | server.listen(port, '127.0.0.1') only — never 0.0.0.0; integration test asserts server.address().address === '127.0.0.1' (HOST-02, acceptance #10) |
| T-02-auth | Tampering | server.ts POST /annotation | mitigate | checkToken gate before any write; 401 {ok:false} on miss; integration test asserts no-token -> 401 (HOST-05) |
| T-02-cors-readable | Information Disclosure | server.ts setCorsHeaders | mitigate | CORS headers on every response so the browser can read 401/413 bodies (no silent "host down"); token remains the real gate (HOST-10, Pitfall 6) |
| T-02-dos | Denial of Service | server.ts readBody mapping | mitigate | readBody 12 MB cap surfaced as 413; req.destroy() on overflow (HOST-11) |
| T-02-info-status | Information Disclosure | server.ts /status | mitigate | /status returns app/version/name/root/notesDir/origins only — never the token (HOST-04, D-06) |
| T-02-SC | Tampering | npm installs | accept | Zero new packages installed in this plan |
</threat_model>
