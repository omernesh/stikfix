<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser page -> POST /annotation | Any local web page that guesses the port can attempt to write files; untrusted input crosses here |
| payload field -> filesystem path | A crafted notes/target path could try to escape --root |
| payload body -> host memory | A large base64 screenshot body could exhaust memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-auth | Tampering | checkToken (security.ts) | mitigate | X-Stickyfix-Token validated with crypto.timingSafeEqual (length-checked first); unit test asserts missing/wrong -> false, exact -> true (HOST-05) |
| T-02-traversal | Tampering | isInsideDir (security.ts), resolveConfig notes-dir guard (config.ts) | mitigate | path.resolve + startsWith(root + sep); resolveConfig rejects notes-dir outside root; unit test asserts ../ and rootfoo-prefix rejected (HOST-09) |
| T-02-dos | Denial of Service | readBody (security.ts) | mitigate | 12 MB hard cap; req.destroy() before reject; unit test asserts over-cap -> statusCode 413 (HOST-11) |
| T-02-timing | Information Disclosure | checkToken (security.ts) | mitigate | timingSafeEqual constant-time comparison, length check before compare (HOST-05) |
| T-02-SC | Tampering | npm installs | accept | Phase 2 installs zero new packages; yaml@2.9.0 already audited [OK] in RESEARCH Package Legitimacy Audit |
</threat_model>
