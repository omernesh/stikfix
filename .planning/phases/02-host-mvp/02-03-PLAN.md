<objective>
Replace the Phase 1 `spawnSync` smoke test with a `spawn` + `readline` smoke test that drives
the now-long-running Phase 2 server: it reads the startup JSON line, probes `GET /status`,
POSTs a token-authenticated free-mode annotation, asserts the `.md` lands on disk, asserts a
no-token POST returns 401, then kills the child — so `npm run check` proves the host boots
end-to-end without hanging.

Purpose: After Plan 02 the server runs indefinitely, so the existing `spawnSync` smoke test would
hang `npm run check` forever (RESEARCH Pitfall 2). This plan makes the phase gate green and gives
a real boot-to-write end-to-end proof on top of the unit/integration tests.
Output: rewritten `scripts/host-smoke-test.mjs`.
</objective>
