<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web origin → extension | A hostile page may try to obtain the token or invoke the native host |
| native channel → SW | The native host's stdio response crosses into SW-persisted credential storage |
| Chrome → native host process | Chrome spawns the native host via the registered manifest |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Spoofing / Info Disclosure | pairing token over native messaging | mitigate | sendNativeMessage/connectNative are extension-context-only APIs (RESEARCH security proof); manifest allowed_origins pinned to the exact extension ID; web pages cannot call them — verified by ONB-03 grep (no native API in content scripts) |
| T-09-07 | Elevation | native host process | mitigate | native-host.ts opens NO port and runs NO HTTP server; it only reads the token/config files and answers GET_TOKEN, then exits (Pitfall 3); SW-as-sole-HTTP-client invariant preserved |
| T-09-08 | Tampering | origin-from-tab invariant | mitigate | handleSendAnnotation + origin derivation from chrome.tabs.get untouched (REGRESSION guard); pairing is additive |
| T-09-09 | Information Disclosure | token at rest | accept | token persisted in chrome.storage.local (same store the manual token field already uses — Phase 3 posture, no new exposure) |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No runtime package added; only optional esbuild devDependency (build tool). RESEARCH Package Legitimacy Audit: existing deps [OK]; no new runtime dep — no install checkpoint required |
</threat_model>
