  <behavior>
    - encodeNativeMessage(obj): returns a Buffer whose first 4 bytes are the UTF-8 JSON byte length as UInt32LE, followed by the UTF-8 JSON bytes.
    - decodeNativeMessages(buffer): yields complete messages and returns the unconsumed remainder; given a buffer split mid-message it returns zero messages and the full remainder; given two concatenated frames it returns both objects.
    - Round-trip: decodeNativeMessages(encodeNativeMessage(x)) yields [x] with empty remainder.
    - Chunk reassembly: feeding the encoded buffer one byte at a time through a stateful reader yields exactly one message after the final byte and none before.
    - readNativeMessages/sendNativeMessage are thin wrappers over process.stdin/process.stdout using ONLY Buffer writes (no string writes) — assert sendNativeMessage's produced bytes equal encodeNativeMessage's (inject a fake writable to capture).
  </behavior>
  <action>

Add `sendNativeMessage(msg, out=process.stdout)` writing ONE `out.write(encodeNativeMessage(msg))` Buffer call (never a string — Pitfall 2) and `readNativeMessages(onMessage, inp=process.stdin)` accumulating chunks into a module-local Buffer and draining via decodeNativeMessages, calling process.exit(0) on 'end'. Node builtins only (`node:buffer` implicit). No WXT/Chrome imports. Write host/test/native-host.test.ts covering the behaviors above using the config.test.ts harness shape; inject a fake writable object `{ written: Buffer[]; write(b){...} }` to assert Buffer-only output.
